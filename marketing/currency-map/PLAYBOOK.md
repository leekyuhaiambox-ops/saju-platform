### 2.2 개선 #1 — Sitemap 확장 (즉시 효과 큼)

현재: 루트 1개. 제안: 31개 시·군 + 7개 업종 카테고리 = **38개 URL**. SPA 라우팅이 `?city=수원` 같은 쿼리 파라미터를 지원하면 즉시 적용 가능.

**제안 sitemap.xml** → 별도 파일 `shared/sitemap_currency_map_proposed.xml` 참고.

⚠️ 주의: 제안 URL이 실제 200을 반환해야 함. React 라우터가 쿼리 파라미터를 읽어서 해당 도시로 자동 필터링하면 OK. 안 되면 Firebase Hosting rewrite로 `/sigungu/수원 → /?city=수원` 매핑 필요. **사용자 확인 액션**: Firebase config + React 라우팅 코드 점검.

### 2.4 개선 #3 — SPA 사전 렌더링 (Vite SSG)

루트 페이지가 JS 렌더 전엔 `<div id="root"></div>` 뿐. Googlebot은 JS를 실행하지만 **렌더 큐가 늦고 일부 콘텐츠는 누락**. 한국어 검색에서 네이버봇은 JS 렌더링이 약하므로 더 심각.

**해결**: Vite SSG 도입 (`vite-ssg` 라이브러리). 빌드 시 31개 시·군 페이지 + 메인 페이지를 정적 HTML로 사전 생성. 각 HTML에 `<h1>{시·군명} 지역화폐 가맹점 지도</h1>` + 업종별 가맹점 수 텍스트 삽입.

**효과 추정**: Naver 색인 +300%, Google 색인 시간 -50%.

### 2.5 개선 #4 — JSON-LD 강화

현재 `areaServed`에 10개 도시만. 31개 모두 명시 + 각 도시 페이지에 LocalBusiness 미니 카드 (영업 시간·업종 카테고리) 추가하면 리치 스니펫 가능성 ↑.
