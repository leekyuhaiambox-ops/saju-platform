### 2.2 버그 #1 — Sitemap 즉시 수정 (가장 중요)

**현재 sitemap.xml 문제**:
```xml
<!-- 누락: <loc>https://geoinfomatic.pythonanywhere.com/</loc> -->
<!-- 누락: 가장 중요한 메인 페이지가 sitemap에 없음 -->
<url><loc>https://geoinfomatic.pythonanywhere.com/profile</loc>...</url>
<!-- /profile은 포트폴리오 사이트로 보이는데 메인 서비스(/)는 빠짐 -->
```

**즉시 수정안** → `shared/sitemap_geoinfomatic_fixed.xml` 참고. 핵심: 메인 페이지를 priority 1.0으로 추가, 그 다음 핵심 동선 URL 추가.

```xml
<url>
  <loc>https://geoinfomatic.pythonanywhere.com/</loc>
  <lastmod>2026-05-18</lastmod>
  <changefreq>weekly</changefreq>
  <priority>1.0</priority>
</url>
```

**검증 액션**: Flask `routes.py`에서 sitemap 핸들러 찾아서 메인 URL 추가 → PA 재배포.

### 2.3 개선 #2 — 검색엔진 사이트 인증 (각 5분)

robots.txt에 이미 Googlebot/NaverBot/Yeti/Daumoa/Bingbot 허용되어 있음. 인증 코드만 박으면 됨. Flask `base.html` 또는 `index.html`의 `<head>`에 환경변수 패턴으로 추가:

```html
{% if GOOGLE_SITE_VERIFICATION %}
<meta name="google-site-verification" content="{{ GOOGLE_SITE_VERIFICATION }}" />
{% endif %}
{% if NAVER_SITE_VERIFICATION %}
<meta name="naver-site-verification" content="{{ NAVER_SITE_VERIFICATION }}" />
{% endif %}
{% if BING_SITE_VERIFICATION %}
<meta name="msvalidate.01" content="{{ BING_SITE_VERIFICATION }}" />
{% endif %}
```

WSGI 환경변수에 4개 추가.

**핵심 기능 3가지**

첫째, **도보 또는 대중교통 모드** 선택. 차 없는 신혼은 도보, 차 있는 가족은 대중교통으로 분석하면 됩니다.

둘째, **10~45분 슬라이더**. "이 위치에서 도보 30분 내 도달 가능한 곳"을 지도에 색칠해서 보여줍니다(아이소크론 분석).

셋째, **8종 시설 동시 분석**: 학교, 병원, 마트, 공원, 헬스장, 약국, 도서관, 카페. 학군지·신혼·시니어 어느 페르소나든 자기 우선순위만 켜면 됩니다.

**호갱노노와 가장 다른 점**

호갱노노는 매물이 메인이고 생활권 점수는 부가 정보입니다. 점수도 단순합니다 (학군·교통·생활편의 3개만).

이 도구는 **매물·광고가 아예 없습니다**. 순수하게 생활권만 분석합니다. AI가 결과를 자연어로 요약해줘서 "이 동네는 학군지로 강점이지만 대형마트 접근성은 평균 미만" 같은 인사이트를 줍니다.
