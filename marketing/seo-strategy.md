# SEO 키워드 + 내부 링크 전략

검색엔진(네이버·구글·다음)에서 트래픽을 가져오는 핵심 문서.

---

## 핵심 키워드 우선순위

### Tier 1 — 검색량 매우 큼 (월 10만+) / 경쟁 매우 강

| 키워드 | 월 검색량 | 경쟁도 | 진입 난이도 |
|---|---|---|---|
| 사주 | 50만+ | 매우 강 | 어려움 |
| 운세 | 30만+ | 매우 강 | 어려움 |
| 무료 사주 | 15만+ | 강 | 중간 |
| 사주풀이 | 10만+ | 강 | 중간 |
| 신년운세 | 시즌형 80만 | 매우 강 | 시즌만 가능 |

**전략:** 단독 진입은 어려움. **롱테일 + 백링크 누적** 후 노려봄. 1년 목표.

### Tier 2 — 검색량 큼 (월 1~10만) / 경쟁 중

| 키워드 | 월 검색량 | 경쟁 | 우리 페이지 |
|---|---|---|---|
| 사주 궁합 | 8만 | 중간 | /compatibility |
| 띠별 운세 | 6만 | 중간 | /zodiac |
| 사주 일주 | 5만 | 중간 | /sixty-pillars |
| 일주 풀이 | 3만 | 약함 | /sixty-pillars/{idx} |
| 십신 | 2만 | 약함 | /ten-gods |
| 사주명리 | 4만 | 중간 | / |

**전략:** **이 티어 진입이 1순위 목표.** 3~6개월 안에 1페이지 진입 목표.

### Tier 3 — 롱테일 키워드 (월 100~1만) / 경쟁 약

| 키워드 | 우리 페이지 | 비고 |
|---|---|---|
| 갑자 일주 | /sixty-pillars/0 | 60개 일주 각각 |
| 갑인 일주 | /sixty-pillars/50 | |
| 병오 일주 | /sixty-pillars/42 | |
| 식신 의미 | /ten-gods | |
| 정관 뜻 | /ten-gods | |
| 십이운성 | /twelve-stages | |
| 장생 의미 | /twelve-stages | |
| 입춘 보정 | /basics | |
| 야자시 | /glossary | |
| 24절기 사주 | /basics | |
| 천간지지 | /glossary | |
| 60갑자 순서 | /sixty-pillars | |

**전략:** **무조건 1위 가능. 60+ 키워드를 동시에 점령.**

### Tier 4 — 슈퍼 롱테일 (월 10~500) / 경쟁 거의 없음

키워드 예시:
- "갑자 일주 직업"
- "병오 일주 연애"
- "정관 강한 사주 특징"
- "사주 십신 외우는 법"
- "오행 부족 보완"
- "야자시 출생자 일주"
- "입춘 1월생 띠"

**전략:** **즉시 1위 진입 가능.** 콘텐츠만 있으면 됨.

## 내부 링크 전략 (이미 구현됨, 강화 필요)

### 링크 허브 페이지 = 60갑자 인덱스

`/sixty-pillars`에서 60개 일주 페이지로 다 연결됨. 이게 SEO의 중심.

### 페이지 간 연결 강도

```
홈 (/)
├── 직접 링크: 결과, 사주 기초, 60갑자, 십신, 십이운성, 오행, 궁합, 오늘의 운세, 띠별 운세, 용어 사전
└── 풋터 링크: 모든 페이지

결과 (/result)
├── 일주 상세 (60갑자 해당 페이지)
├── 궁합 페이지
├── 오늘의 운세
└── 다른 사주 보기

60갑자 상세 (/sixty-pillars/N)
├── 이전·다음 일주
├── 60갑자 목록
└── 홈으로

콘텐츠 페이지 (basics, ten-gods, ...)
├── 다른 학습 페이지 (상호 링크)
└── 홈 사주 풀이 CTA
```

### 권장 추가 작업

1. **본문 내 키워드 링크** — 콘텐츠 페이지에서 다른 용어 언급 시 해당 페이지로 링크
   - 예: `basics.html`에서 "십신" 언급 시 → `/ten-gods` 링크 (이미 있음, 더 추가 가능)

2. **관련 일주 추천** — `pillar_detail.html` 하단에 "같은 일간(예: 갑목)의 다른 5개 일주" 자동 노출

3. **태그 페이지** — `/tag/리더십`, `/tag/예술`, `/tag/직장운` 같은 페이지를 만들어 관련 일주·콘텐츠를 묶기

## 메타 데이터 체크리스트

각 페이지가 다음을 충족하는지 확인:

- [ ] `<title>` 키워드 1회, 60자 이내
- [ ] `<meta description>` 키워드 1~2회, 150자 이내
- [ ] `<h1>` 페이지당 1개, 키워드 포함
- [ ] `<h2>` 3개 이상, 키워드 변형 사용
- [ ] JSON-LD Article/Breadcrumb 스키마 (이미 구현됨)
- [ ] OG 이미지 1200x630 (이미 구현됨)
- [ ] canonical URL 정확 (이미 구현됨)

## 네이버 SEO 특화 팁

네이버는 구글과 알고리즘이 완전히 다릅니다.

### 네이버가 좋아하는 것

1. **본문 1500자 이상**
2. **이미지 5장 이상** (alt 텍스트 키워드)
3. **블로그 형식** (네이버 검색 1페이지의 70%가 블로그)
4. **다양한 멀티미디어** (이미지·동영상·표 혼합)
5. **체류 시간 1분 이상**

### 네이버가 싫어하는 것

1. **광고 위주** (광고 3개 이상이면 SEO 페널티)
2. **외부 링크 남발** (5개 이상 자동 페널티)
3. **중복 콘텐츠** (다른 블로그에 같은 내용)
4. **짧은 글** (500자 미만)

### 네이버 검색 1페이지 노리는 법

1. **C-Rank·DIA 알고리즘 이해**
   - C-Rank: 블로그 신뢰도 (꾸준한 게시·전문성)
   - DIA: 사용자 만족도 (체류 시간·이탈률)

2. **블로그 vs 사이트 분리 전략**
   - 사이트 (PythonAnywhere): 도구·계산기 중심
   - 네이버 블로그: 정보·해설 글 중심
   - 블로그 → 사이트로 자연스러운 유도

3. **인플루언서 등록**
   - 네이버 블로그 운영 6개월 + 글 30개 이상 → 인플루언서 신청
   - 인플루언서 글은 검색 상단 우대

## 구글 SEO 특화 팁

### 구글이 좋아하는 것

1. **E-E-A-T** (Experience, Expertise, Authoritativeness, Trustworthiness)
2. **모바일 친화** (이미 구현됨)
3. **빠른 로딩** (Core Web Vitals)
4. **양질의 백링크**
5. **HTTPS** (이미 구현됨)

### 구조화 데이터 충실도

이미 구현된 것:
- WebSite + SearchAction
- Article (콘텐츠 페이지)
- BreadcrumbList
- DefinedTermSet (용어 사전)

**추가 권장:**
- FAQPage (basics·ten-gods 등에 Q&A 섹션 추가하면)
- Person (운영자 소개 시 활용 가능)
- Review (사용자 후기 받으면 활용)

### 백링크 만들기

1. **네이버 블로그 → 사이트** (자가 백링크, 가치 낮음)
2. **티스토리·브런치·미디엄 → 사이트** (가치 중간)
3. **사주 관련 큰 사이트의 게시판 댓글** (가치 약함)
4. **언론사·전문지 인용** (가치 최고, 어렵지만 가능)
5. **위키 백과 한국어 사주 항목** (가치 최고, 출처 등재 시도)

## 측정 도구

1. **Google Search Console** (필수, 무료)
   - 어떤 키워드로 들어왔는지
   - 노출 vs 클릭률
   - 페이지별 색인 상태

2. **Naver Search Advisor** (필수, 무료)
   - 네이버 검색 노출
   - 사이트맵 제출

3. **Google Analytics 4** (필수, 무료)
   - 유입 출처
   - 페이지 체류 시간
   - 사주 풀이 완료율

4. **Ahrefs / Semrush** (유료, 월 10만원~)
   - 경쟁 사이트 분석
   - 백링크 추적
   - 초기엔 무료 도구로 충분
